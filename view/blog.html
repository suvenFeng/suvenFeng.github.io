网站如何解决图片过大加载慢的问题？

1月 1, 2014 by 冯鹏

图片过大加载慢？而不是图片太多加载慢？.

首先要尽可能的压缩，看用户的忍受程度。jpg 可以用很多方法压缩，png 推荐使用 TinyPNG 工具（不过正文图片，显然要选择 jpg 格式的）.

1，不太“在乎”用户体验的省事方法: 图片保存成渐进式的; 加载会慢慢变清晰，而不是从上往下依次加载，然后放在 CDN，设置缓存之类.

2: 比较“在乎”用户体验的高端方案：

Heading

判断用户的设备（主要用在移动端）、网络等，分别加载不同质量的图片（例如高端 iPhone wifi 情况下，就可以加载双倍高清图等，蜂窝网络下面，就加载个单倍或者有损压缩过的）.

Sub-heading

或者先加载低质量的图片，让浏览者可以看到，然后再在后台加载更高清的，等加载完了，浏览者还在观看，就插入替换掉。.

Example code block

或者先加载低质量小图片列表，然后让用户点击，触发类似 fancybox 的效果，弹窗出现大图片。.

Sub-heading

或者利用资源预加载（三个 HTML5 不常见特性简介）当用户还没打开的时候，就开始加载。 还有好多思路，后面想到再补充。

另外的解决办法

11月 23, 2013 by SUVEN

直接一开始用压缩过的体积小的图，等加载完毕后，再用高清无码图来替换掉.

高清无码的图片随便怎么命名，但是压缩过的命名有要求， 那就是必须在原图的名称前面加上相同的便于分辨的标识。比如.

高清图1命名为wozuishuai.jpg 那么相应的压缩图就叫wozuishuai_min.jpg (_min这个随你取名，但是要一致，不能一个叫_min,下一个就成了_small) 然后，
